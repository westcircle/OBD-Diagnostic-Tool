# main_cli.py 実車確認チェックリスト

このチェックリストは、`main_cli.py` を使って実車で OBD 接続から診断まで確認するための手順です。  
CLI（ターミナル）での操作を前提にしています。

---

## 事前準備

- 車両のイグニッションを ON（またはエンジン始動）にする
- OBD 機器（ELM327 / OBDLink SX）を車両と PC に接続する
- ターミナルで作業フォルダへ移動する

```bash
cd /d C:\Users\User\Desktop\car_diagnosis_ai
python main_cli.py
```

---

## 1. OBD機器接続

- 操作: OBD機器を車両OBDポートとPCに接続
- 成功の目安: 機器の接続ランプが点灯し、PC側でCOMポートが認識される
- 失敗時に見るポイント: ケーブル抜け、別USBポート、ドライバ未認識、他アプリがCOMポート使用中

## 2. main_cli.py 起動

- 操作: `python main_cli.py` を実行
- 成功の目安: メニューと「現在状態」が表示される
- 失敗時に見るポイント: Python実行パス、作業ディレクトリ、エラーメッセージ内容

## 3. OBD接続

- 操作: メニューで `1. OBD接続` を選択
- 成功の目安: `ECU接続成功` のログが出る
- 失敗時に見るポイント: 接続失敗理由（NO DATA / UNABLE TO CONNECT / COM使用中 / タイムアウト など）

## 4. VIN取得

- 操作: メニューで `3. VIN取得` を選択
- 成功の目安: VIN文字列（通常17桁）が表示される
- 失敗時に見るポイント: 接続済みか、車両がVIN応答に対応しているか、通信状態

## 5. DTC読取

- 操作: メニューで `4. DTC読取` を選択
- 成功の目安: `DTC なし` または `Pxxxx` 形式のコードが表示される
- 失敗時に見るポイント: ECU応答有無、接続ログ、キーON状態

## 6. 基本PID表示

- 操作: メニューで `6. 基本PID表示` を選択
- 成功の目安: RPM / ECT / MAF / SPEED / IAT / THROTTLE が数値または `N/A` で表示される
- 失敗時に見るポイント: すべて `N/A` の場合は通信状態や車両側PID対応状況

## 7. ライブデータ表示

- 操作: メニューで `7. ライブデータ表示` を選択
- 成功の目安: 1秒ごとに基本PIDが更新表示される
- 失敗時に見るポイント: 値が更新されない場合は接続状態、終了方法は Enter または Ctrl+C

## 8. OBD診断実行

- 操作: メニューで `8. OBD診断実行` を選択し、入力項目（メーカー/車種/年式/走行距離/症状）を入力
- 成功の目安: 診断結果（原因候補・確認手順・緊急度）が表示される
- 失敗時に見るポイント: DTCが取得できているか、入力値の空欄、ログの警告

## 9. レポート保存確認

- 操作: OBD診断実行後に保存メッセージを確認し、`reports` フォルダを確認
- 成功の目安: `report_*.txt` が生成されている
- 失敗時に見るポイント: フォルダ権限、保存先パス、エラーログ

## 10. 履歴CSV保存確認

- 操作: OBD診断実行後に `logs/history.csv` を確認
- 成功の目安: 1実行につき1行追加される（日時、VIN、DTC一覧など）
- 失敗時に見るポイント: `logs` フォルダ有無、CSV書き込み権限、診断処理が完了しているか

## 11. DTC消去確認

- 操作: 診断後の確認で `DTCを消去しますか？ [y/N]:` に `y` を入力
- 成功の目安: `DTC消去実行` ログと消去コマンド送信ログが出る
- 失敗時に見るポイント: 接続状態、車両条件（キーON/エンジン状態）、ECU側制約

## 12. 消去後再確認

- 操作: DTC消去後の自動再確認結果を確認
- 成功の目安: `消去後DTCなし` と表示される
- 失敗時に見るポイント: `まだDTCあり` の場合は恒久故障や再発条件を疑い、再点検

---

## 補足

- このツールは診断支援用です。最終判断は実車確認・追加点検を前提にしてください。
- DTC消去は、記録を残した上で実施してください。
