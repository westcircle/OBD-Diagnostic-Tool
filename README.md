# OBD Diagnostic Tool

Pythonで作成している、自動車向けの簡易OBD診断ツールです。  
市販のOBDアダプタを使いながら、車両情報の取得、故障コードの確認、簡易的な状態把握を行うことを目的に開発しています。

このプロジェクトは、既製品を使うだけではなく、**自分で診断の流れを理解しながら育てていくこと**を重視しています。

---

## このプロジェクトでできること

- OBDアダプタ経由で車両へ接続
- VINの取得
- WMI / VDS を使ったメーカー・車両情報の簡易推定
- DTC（故障コード）の読み取り
- DTC説明文の表示
- RPM、水温、MAFなどのPID読み取り
- 正常値辞書との比較に向けた基礎構造
- シミュレーターを使った開発・検証

---

## 主なファイル

- `main.py`  
  メイン処理です。OBD通信、VIN取得、PID読取、DTC表示などを行います。

- `main_cli.py`  
  CLIで扱いやすい実行用コードです。

- `simulator.py`  
  実車がなくても開発やテストがしやすいようにしたシミュレーターです。

- `generate_dtc.py`  
  DTC説明の自動生成用スクリプトです。

- `merge_dtc.py`  
  複数のDTC辞書を統合するためのスクリプトです。

- `register_vds.py`  
  VDS情報の追加・登録補助用スクリプトです。

- `dtc_database.json`  
  ベースとなるDTC辞書データです。

- `dtc_auto_generated.json`  
  自動生成したDTC説明データです。

- `dtc_description.json`  
  手作業で整えた説明データです。

- `dtc_merged.json`  
  統合済みのDTC辞書です。

- `manufacturers.json`  
  メーカー情報の辞書です。

- `wmi_map.json`  
  WMIからメーカーを判定するための辞書です。

- `vds_map.json`  
  VDSから型式・エンジン候補などを推定するための辞書です。

- `pid_normal.json`  
  PIDの正常値判定に使うための辞書です。

- `obd_log.csv`  
  通信や診断のログ保存用ファイルです。

- `diagnostic_log.txt`  
  デバッグ確認用のログです。
